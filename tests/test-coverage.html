AuditVerse Coverage Calculation Test

Test Scenario Explanation

The header statistics now display:

  • Total Audits: Number of audit entities that assess the filtered risks (i.e., audits that have "assessed_by" relationships with filtered risks)
  • Total Risks: Number of risk entities that meet all filter criteria
  • Coverage %: Percentage calculated as (Risks linked to audits) / (Total filtered risks) × 100

Filters Applied: The calculation respects all header filters:

  • Audits filter - only includes selected audits
  • Business Units filter - only includes risks owned by selected units
  • Standards filter - only includes risks requiring selected standards
  • Risk Type filter - only includes risks of selected categories

Manual Test Steps

  1. Open the application at http://localhost:5176/
  2. Load the comprehensive sample data
  3. Observe the initial coverage percentage in the header (should reflect all risks)
  4. Use the audit dropdown filter - deselect some audits
    • Coverage should update to show only risks linked to selected audits
  5. Use the business units dropdown filter - deselect some units
    • Coverage should update to show only risks owned by selected units AND linked to selected audits
  6. Use the standards dropdown filter - deselect some standards
    • Coverage should update to show only risks requiring selected standards AND matching other filters
  7. Use the risk type dropdown filter - deselect some risk types
    • Coverage should update to show only risks of selected categories AND matching other filters

Expected Behavior

With All Filters Selected (Default):

Coverage = (All risks with audit relationships) / (All risks) × 100%

With Some Audits Deselected:

Coverage = (Risks linked to selected audits) / (All risks matching other filters) × 100%

With Multiple Filters Applied:

Coverage = (Filtered risks that have audit relationships with selected audits) / (All risks matching all filters) × 100%

Implementation Summary

The coverage calculation has been updated in src/js/main.js with an enhanced calculateFilteredCoverage() function that:

  • Filters audits based on the audit dropdown selection
  • Filters risks based on selected risk types, business units, and standards
  • Counts audits that have relationships with filtered risks (displays as "Total Audits")
  • Counts all filtered risks (displays as "Total Risks")
  • Counts how many filtered risks are linked to filtered audits (for coverage calculation)
  • Calculates the coverage percentage as: (risks linked to audits / total filtered risks) × 100
  • Updates all three values (Total Audits, Total Risks, Coverage %) automatically when any filter changes

Key Changes:

  • "Total Audits" now shows the count of audits that assess the filtered risks (not all audits)
  • "Total Risks" shows the count of risks that meet all filter criteria
  • "Coverage %" shows what percentage of filtered risks are covered by audits
